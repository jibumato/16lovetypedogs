# 16わんこ SNS運用チーム 体制 / Team Operating Model

「何を作るか（戦略・台本）」は `INSTAGRAM-PLAYBOOK.md` と `GROWTH-PLAYBOOK.md` に、
「どう書くか（コピー・タグ）」は `sns-captions.md` にあります。
このドキュメントは **「誰が・いつ・どう回すか」=運用チームの編成と稼働フロー** を定義します。

> チームは **AIエージェント** でも **人** でも、両者の混成でも回せる設計です。
> 1人運用の場合は「1人がロールを順番にかぶる」チェックリストとして使ってください。

---

## 1. チーム編成（6ロール）

| # | ロール | 一言ミッション | 主なインプット | 主なアウトプット |
|---|--------|----------------|----------------|------------------|
| 1 | **編集長 / Lead** | 方針決定・カレンダー管理・最終OK | KPI・前週の数字 | 週次の投稿割当・GO/NG判断 |
| 2 | **企画 / Planner** | ネタ出し・台本選定 | `INSTAGRAM-PLAYBOOK.md` ①台本A〜E | 各投稿の台本＋使う素材の指定 |
| 3 | **制作 / Creator** | リール・カルーセル組み立て | 指定された `ad-*.png` 素材 | 投稿用ビジュアル（動画/画像） |
| 4 | **コピー / Writer** | キャプション・ハッシュタグ・多言語 | `sns-captions.md` | 各言語のキャプション完成稿 |
| 5 | **配信・接客 / Community** | 投稿・ストーリー・コメント返信 | 完成稿・ストーリーテンプレ | 公開済み投稿・UGCループ |
| 6 | **分析 / Analyst** | 数字を読んで次週へ反映 | GA4 / Search Console / IGインサイト | 週次レポート＋改善提案 |

### 各ロールの責任範囲（DoD = Definition of Done）

- **編集長**：週初に7枠（曜日×形式）を確定し、各枠にPlannerをアサイン。公開前の最終チェック（売り込み比率90:10／CTAが無料診断か／タグ付け・保存導線があるか）。
- **企画**：各投稿に「台本ID（A〜E）＋テーマ（例: INFP）＋使う素材ファイル名＋CTA」を1行で指定。迷ったら主力の**台本A（あるある）**を選ぶ。
- **制作**：フックは**最初の1秒**に最大文字。素材は `ad-aruaru-{type}.png` / `ad-aisho-{pair}.png` / `ad-types-chart.png` を流用。横展開時は多言語版 `-en/-ko/-zh/-tw` に差し替え。
- **コピー**：`sns-captions.md` をベースに、ハッシュタグは大中小ミックスで8〜15個。CTAは常に「プロフィールから無料診断」。
- **配信・接客**：平日21〜23時／休日昼〜夜に公開。フィード→ストーリーで再告知。コメントは**わんこキャラの温かいトーン**で必ず返信（カフェ的常連づくり）。
- **分析**：先行指標（保存・シェア・プロフィール訪問→リンククリック）を最優先で確認。「伸びた投稿／伸びなかった投稿」を各1つ特定して次週へ。

## 2. 週次オペレーション（標準サイクル）

```
月  編集長: 今週7枠を確定 → 各枠にPlannerアサイン / Analyst: 先週レポート共有
火  企画→制作→コピー: 週前半3本を仕込み（あるある・共感・クイズ）
水  配信: 平日投稿＋ストーリー1〜2本 / 接客: コメント返信
木  企画→制作: 週末の拡散系（相性・ランキング）を仕込み
金  配信: 神相性リール＋ig-story-tag で友達タグ付け誘発
土日 配信: ランキング(保存)・相性(タグ付け) / 接客: 結果シェア募集(ig-story-result)でUGC
```

**理想ペース**：1日1フィード（orリール）＋ストーリー1〜2本。
**現実ライン**：週4〜5本でも**継続最優先**。落ちるくらいなら台本E(共感ひとこと)で軽く繋ぐ。

> 曜日割当の詳細テンプレは `INSTAGRAM-PLAYBOOK.md` ②「30日投稿カレンダー」を流用。

## 3. ハンドオフ・フロー（1投稿あたり）

```
編集長(枠決定)
   └─▶ 企画(台本ID＋素材＋CTAを指定)
          └─▶ 制作(ビジュアル) ─┐
          └─▶ コピー(各言語文) ─┴─▶ 編集長(最終チェック)
                                        └─▶ 配信(公開＋ストーリー)
                                               └─▶ 接客(コメント返信・UGC回収)
                                                      └─▶ 分析(翌週へ反映)
```

制作とコピーは**並行**で進めてOK。ボトルネックは「素材選定の迷い」なので、
企画が**素材ファイル名まで指定**することで以降を詰まらせない。

## 4. 言語別アカウントの分担

多言語素材（`-en/-ko/-zh/-tw`）があるため、アカウントを言語別に分離して運用します。

| アカウント | 優先度 | 理由 | 担当の置き方 |
|------------|--------|------|--------------|
| 繁体字 (tw) / 簡体字 (zh) | ★最優先 | アナリティクス上**中国語圏のエンゲージが最高** | コピー＋接客を厚めに |
| 日本語 (ja) | ★主力 | 素材・台本が最も揃う | フル6ロールで運用 |
| 英語 (en) / 韓国語 (ko) | ◯横展開 | 日本語投稿をキャプション差し替えで再利用 | 1人が制作流用＋翻訳 |

> 横展開の鉄則：**1本作ったら多言語キャプションに差し替えて各アカウントへ**（制作コストを増やさない）。

## 5. KPI と振り返り（Analyst担当）

**先行指標（毎週見る／売上より先に）**
- リーチ ／ **保存数** ／ **シェア数**（＝拡散の燃料）
- プロフィールアクセス → **リンククリック**（＝無料診断への送客）
- 結果シェア（UGC）件数

**遅行指標（月次）**
- GA4の診断開始数 → `begin_checkout` → `purchase`（計測は実装済み・タグ `G-C3W7FBQRCD`）
- Search Console の表示回数・クリック・掲載順位

**週次レポート雛形（3行でOK）**
1. 伸びた投稿1つ＝何が効いたか（フック/素材/CTA）→ 来週**再利用・リール化**
2. 伸びなかった投稿1つ＝仮説 → 来週**フックだけ差し替えて再挑戦**
3. 来週の重点1つ（例: tw account の相性ネタを増やす）

---

## 6. ガードレール（やらないこと）

- 毎投稿での売り込み（売り込みは10%まで・入口は常に無料診断）
- フォロー&アンフォロー、無関係なバズ便乗
- コメント無視（接客=常連づくりが拡散の土台）
- 多言語で**個別に作り込む**（横展開＝差し替えで回す）

## 7. AIエージェントで回す場合（運用メモ）

各ロールはエージェントに割り当て可能です。推奨の依頼単位：

- **Planner エージェント**：「今週の7枠を、台本A〜EとMBTIタイプ16種から重複なく割当てて。各枠は『台本ID＋タイプ＋素材ファイル名＋CTA』の1行で」
- **Writer エージェント**：「この台本の日本語キャプション＋ハッシュタグ8〜15個を `sns-captions.md` のトーンで。続けてen/ko/zh/twへ差し替え版も」
- **Analyst エージェント**：「貼り付けたIGインサイト/GA4の数字から、伸びた1本・伸びなかった1本・来週の重点1つを3行で」

> 制作（実際の動画書き出し）と配信（IG投稿）・接客（返信）は、現状プラットフォーム操作が要るため**人の手**が前提。
> AIは「企画・コピー・分析」を高速化し、人は「制作・配信・接客」に集中する分担が現実的です。
